# TripTally 微信小程序项目指南

## 项目概述
TripTally 是一个面向5人以下小团队的旅行记账微信小程序。团队成员通过创建或加入"出行密码"（房间号）共享账本，记录旅行中的公共花销，并自动计算AA分摊。

## 核心功能需求

### 1. 房间管理
- 创建房间：生成6位数字出行密码（房间号）
- 加入房间：输入出行密码加入现有房间
- 房间人数限制：最多5人
- 离开房间：成员可主动退出

### 2. 记账功能
- 添加支出：输入金额、分类、描述、支付者
- 自动记录：支出日期、创建时间自动记录
- 修改记录：编辑或删除已有支出
- 参与人选择：选择参与本次AA的成员（默认全选）

### 3. AA自动计算
- 实时计算：每次账目变动后自动重新计算
- 计算逻辑：总支出 ÷ 参与人数 = 人均应付
- 结果显示：每人应收/应付金额，正数为应收，负数为应付
- 结算建议：生成最少转账次数的结算方案

### 4. 数据共享与同步
- 实时同步：房间内所有成员看到相同的账目数据
- 数据安全：只有房间成员可查看和修改数据
- 离线支持：短暂离线后能同步最新数据

### 5. 数据安全与防误触
- 防误触设计：已写入的款项修改需要二次确认
- 滑动删除确认：删除支出记录时需要滑动确认操作
- 操作权限控制：仅支付者或房间创建者可修改支出记录
- 历史版本管理：定时保存历史数据，支持数据回滚
- 数据备份：支持查看历史版本和恢复到指定时间点

## 非功能需求
1. **性能**：账目计算和同步响应时间 < 2秒
2. **可靠性**：数据不丢失，支持冲突解决
3. **安全性**：房间密码有一定随机性，防止轻易猜测
4. **可维护性**：代码结构清晰，便于后续功能扩展

## 项目架构

### 前端页面结构
```
pages/
├── index/                    # 首页/入口页（现有，需改造）
├── create-room/             # 创建房间页
├── join-room/              # 加入房间页
├── room-detail/            # 房间详情页（主记账界面）
├── add-expense/            # 添加支出页
├── expense-detail/         # 支出详情页
├── settlement/             # 结算页
└── history/                # 历史记录页
```

### 云数据库集合
1. `rooms` - 房间信息
2. `room_members` - 房间成员关系（包含用户ID和昵称）
3. `expenses` - 支出记录
4. `settlements` - 结算记录
5. `expense_history` - 支出历史版本（用于数据恢复）

注：已移除`users`集合，使用匿名用户ID系统，用户ID由前端生成并存储。

### 核心云函数（已实现）
- `createRoom` - 创建房间并生成唯一房间码（使用匿名用户ID）
- `joinRoom` - 加入房间验证和成员管理（使用匿名用户ID）
- `addExpense` - 添加支出并触发AA计算（使用匿名用户ID）
- `calculateSettlement` - 计算AA结算方案
- `syncData` - 数据同步和冲突解决
- `createHistoryBackup` - 创建历史数据备份（使用匿名用户ID）
- `getExpenseHistory` - 获取支出历史版本（使用匿名用户ID）
- `restoreFromHistory` - 从历史版本恢复数据（使用匿名用户ID）

注：已移除微信登录功能，使用匿名用户ID系统，用户ID由前端生成并传递给云函数。

## 开发规范

### 代码风格
- 使用ES6+语法
- 组件命名使用小写字母和连字符（如 `expense-item`）
- 页面文件结构：index.js, index.wxml, index.wxss, index.json
