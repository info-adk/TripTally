# TripTally

多人 AA 记账微信小程序，专为 5 人以下小团队旅行记账设计。

## 功能特性

### 房间管理
- **创建房间**：输入房间名称，快速创建专属账本，自动生成 6 位数字房间码
- **加入房间**：输入房间码即可加入，与队友共享账本
- **离开房间**：成员可随时退出，房间继续由其他成员使用
- **我的房间**：快速访问最近加入的房间

### 记账功能
- **添加支出**：输入金额、选择分类（餐饮/交通/住宿/购物/娱乐/其他）、添加描述
- **选择支付者**：谁是这笔钱的付款人
- **选择参与人**：默认全员 AA，也可自定义参与人员
- **编辑/删除**：支出记录支持修改和删除（仅支付者或房间创建者可操作）
- **删除确认**：滑动删除需二次确认，防止误操作

### AA 自动计算
- **实时计算**：每次账目变动自动重新计算
- **精确计算**：金额精确到小数点后 2 位
- **结算方案**：生成最少转账次数的结算建议，告诉谁该给谁多少钱
- **个人明细**：清晰展示每人已付、应付和差额

### 数据同步
- **实时同步**：房间内所有成员实时看到相同数据
- **历史记录**：支持查看所有操作历史
- **数据恢复**：可回滚到任意历史版本

### 用户系统
- **昵称入驻**：首次使用输入昵称即可，无需复杂注册
- **同昵称识别**：同一房间内相同昵称的用户被视为同一人（适合情侣/家人共享）

## 技术栈

- **平台**：微信小程序
- **数据库**：微信云开发（云数据库）
- **云函数**：10+ 个云函数处理核心业务逻辑
- **渲染引擎**：Skyline
- **组件框架**：glass-easel

## 数据库集合

| 集合 | 说明 |
|------|------|
| `rooms` | 房间信息（名称、密码、创建者等） |
| `room_members` | 房间成员（昵称、余额等） |
| `expenses` | 支出记录 |
| `settlements` | 结算记录 |
| `expense_history` | 操作历史（用于数据恢复） |

## 快速开始

1. 克隆项目到本地
2. 使用微信开发者工具打开项目
3. 配置云开发环境
4. 上传所有云函数到云开发
5. 在云开发控制台创建所需的数据库集合
6. 运行项目

## 开发说明

### 云函数部署
在微信开发者工具中，右键点击 `cloudfunctions` 目录，选择「上传并部署所有云函数」。

### 数据库权限
建议设置：
- `rooms`：所有用户可读，仅创建者可写
- `room_members`：所有用户可读，仅创建者可写
- `expenses`：所有用户可读，房间成员可写
- `settlements`：所有用户可读
- `expense_history`：所有用户可读，房间成员可写

### 代码规范
- 金额计算统一使用 `Math.round(amount * 100) / 100` 保留 2 位小数
- 云函数返回格式：`{ success: boolean, message: string, data?: any }`
- 用户标识使用 `userName`（昵称），而非用户 ID
